데이터베이스의 이해

1. 데이터베이스 시스템의 목적

(1) 데이터의 종속

데이터의 독립성은 크게 논리적 데이터 독립성과 물리적 데이터 독립성으로 구분
논리적 데이터 독립성이란 데이터의 논리적 구조가 변하더라도 프로그램 구조가 변경되지 않은 것
물리적 데이터 독립성이란 보조기억장치와 같이 파일과 관련된 물리적인 시스템 구조가 변경되더라도 프로그램은 그대로 유지
DBMS는 데이터를 프로그램과 완전히 독립시킴으로써 종속으로 인한 문제를 배제시킨다

(2) 데이터 중복

데이터가 여러 파일에 중복되어 저장되는 문제
☞일관성의 문제, 보안성의 문제, 경제성의 문제

(3) 데이터의 무결성 훼손

데이터의 무결성 훼손은 하나의 사실을 표시하는 두 개 이상의 중복된 데이터가 서로 일치하지 않는 경우
저당된 데이터 값이 일정한 형식의 조건들을 만족하지 않는 부정확한 데이터 저장으로 발생

(4) 동시 접근 이상

여러 사용자가 동시에 사용하는 시스템에서 한 데이터에 대한 수정 요구가 동시에 발생한 경우 비정상적인 데이터 수정 발생

2. 데이터베이스 관리 시스템의 특징

(1) 프로그램과 데이터의 독립성 및 추상화

프로그램은 DBMS에 필요 데이터를 데이터의 의미만으로 요청할 수 있도록 데이터 사용과 데이터 관리가 분리된다
이러한 특성을 프로그램-데이터 독립성이라고 한다
사용자가 데이터의 의미만으로 접그날 수 있도록 데이터에 대한 개념적인 표현을 제공
이러한 개념은 데이터 구조화를 위한 데이터 추상화라는 작업으로 이루어진다

(2) 자기 기술성

데이터에 대한 정의나 의미까지 관리하는 자기 기술성(self-describing)
DBMS는 데이터베이스에 속하는 각 파일의 구조, 각 데이터 항목의 타입과 저장 형식, 데이터의 다양한 제약조건 등의 메타데이터를 포함

(3) 다중 뷰

사용자의 역할과 권한에 맞는 데이터에 접근할 수 있도록 필요한 부분만 추출해서 제공할 수 있는 뷰 기능

(4) 다수 사용자 요청 처리

여러 사용자가 동시에 데이터베이스에 접근할 수 있는 기능을 제공
DBMS에는 단일 논리적인 작업을 수행하는 일련의 데이터베이스 명령의 집합인 트랜잭션과 동시성 제어 기법이라는 기능 내장
이를 통해 다수 사용자가 동일한 데이터를 동시에 변경해도 데이터의 일관성 유지

3. 데이터베이스 관리 시스템의 구조

(1) 데이터 추상화

데이터 추상화와 데이터 독립성을 확보하기 위한 3단계 구조
☞ 내부단계 → 가장 낮은 추상화 단계로 내부 스키마에 의해 기술
☞ (개념-내부 사상) ☞ 개념단계 → 개념 스키마. 데이터베이스에 무엇이 저장되어 있는지와 데이터 간의 관계만 기술
☞ (외부-개념 사상) ☞ 외부단계 → 추상화의 최상위 단계로 외부 스키마, 즉 뷰(view)에 의해 기술. 사용자가 관심 갖는 데이터베이스 일부분만 기술하고 다른 부분은 은폐

(2) 단계 간 사상

☞ 외부-개념 사상 → 외부 스키마(뷰)와 개념 스키마 간의 대응 관계를 정의
개념 스키마에 변화가 생기더라도 그 변화를 외부-개념 사상에만 반영시켜 주면 외부 스키마에 아무런 영향X
논리적 데이터 독립성이 확보

☞ 개념-내부 사상 → 개념 단계의 데이터 스키마가 디스크 내의 내부 필드와 어떻게 대응하는가를 정의
물리적 변화가 발생해도 그 변화를 개념-내부 사상에만 반영시켜 주면 개념 스키마는 아무런 영향도 미치지 않는다
물리적 데이터 독립성 확보

4. 데이터베이스 언어

(1) 데이터 정의 언어(DDL)

스키마를 정의하기 위한
데이터베이스 객체를 생성, 수정, 삭제하기 위한 언어
DDL 명령의 결과는 메타데이터를 저장하는 시스템 카탈로그(또는 데이터 사전)에서 관리

(2) 데이터 조작 언어(DML)

데이터 갱신과 질의를 표현하기 위한
구조화된 데이터에 사용자가 집근 및 조작할 수 있도록 지원하는 언어(검색, 삽입, 수정, 삭제)
